## 🔴 오늘 추가 피드백 — 주제 집중 요구

> 출처: 팀 채널 공유 (2026-06-09 오후)

### 핵심 지적
**"펫로스 증후군 극복 시스템이 없다"** — 기능들이 뒤섞여 앱의 목적이 불분명.

### 지적 항목별 실제 상태 및 대응

| 지적 | 실제 상태 | 대응 |
|------|-----------|------|
| RAG 어디에 어떻게 쓰이나? | comfort·mission·funeral·vet_protocol 4카테고리 실사용 중 | 발표에서 RAG → 기능 연결 명시 필요 |
| 감정 체크인 후 뭘 해주나? | risk_level → 회복 게이트 → 미션 추천 → 1인칭 편지 해금 흐름 구현됨 | **이 흐름이 발표에 안 보이는 게 문제** — 시나리오에 명시 |
| 일상복귀 미션이 구체적으로 없다? | 미션 추천 API·화면·30개 풀 완성됨 | 발표 시나리오에서 강조 |
| 사용자 패턴 분석 없음 (영상 재생횟수·접속빈도) | 미구현 | build_report에 video_play_count·session_count 추가 검토 |
| 투약/검진 기록은 왜 있나? | 시한부 진입점 서비스와 맥락 안 맞음 | **발표 범위에서 제외**, 화면 노출 안 함 |
| 주제가 너무 넓음 | 시한부 케어·펫로스 극복·생전기록·영상 뒤섞임 | 핵심 흐름 하나로 집중 |

### 핵심 흐름 재정의 (발표 기준)

```
시한부 선고
  → 생전 기억 축적 (버킷리스트·사진)
  → 이별
  → 감정 체크인 (risk_level 측정)
  → 회복 게이트 통과 (content_unlocked)
  → 일상복귀 미션 추천
  → 추모 메시지·TTS·1인칭 편지 해금
  → 영상으로 기억 보존
```

### 발표에서 제외할 것
- 투약·검진 기록 화면
- 시한부 증상 안내 (발표 흐름에서 부가 정보로 격하)
