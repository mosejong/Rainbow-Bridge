## 🔗 연구 권고 → 앱 기능 매핑

> 두 출처(학술논문 + 실무 기관)가 동일한 방법론을 제시 → **검증된 처방**임을 발표에서 강조.

| 단계 | 연구 권고 (공통) | 레인보우 브릿지 기능 | 구현 상태 |
|------|----------------|-------------------|---------|
| 1 | 자신의 감정을 솔직하게 표현 | 감정 체크인 (점수·메모 기록) | ✅ |
| 2 | 반려동물과의 좋았던 추억 상기, 사진첩 | 생전 기록 (버킷리스트·일기·사진·타임라인) | 🟡 일부 구현 |
| 3 | 사랑했던 사실 변하지 않음 / 현재 삶 살아가기 | 일상복귀 미션 + 추모 메시지 ("그도 네 행복을 바란다") | ✅ |
| 4 | 죽음을 기념하는 의식 (장례·제사·기념일) | 장례 안내 + TTS·편지·영상 추모 | ✅ |
| 5 | 유품 순차 정리 / 심리적 충격 완화 | 회복 게이트 (단계별 콘텐츠 해금) | ✅ |
| 6 | 입양 때부터 이별 준비 / 능동적 치유 | 시한부 케어 진입점 (선고 직후 앱 시작) | ✅ |
| 7 | 다른 반려동물 들이기 (신중하게) | **앱 범위 밖** — 발표에서 명시 | — |

## 📊 제안 PPT 흐름

```
1. 펫로스 증후군이란? (정의·국내 반려동물 1500만 가구 규모)
2. 극복 방법론 — 학술논문 + 대한동물약국협회 7단계
3. 7단계 → 레인보우 브릿지 기능 매핑 (한 슬라이드)
4. 실제 앱 흐름 시연
   시한부 선고 → 생전 기록 → 사후 추모 → 일상복귀 미션 → 정량 분석
5. 정량적 입증 계획
   감정추이 그래프 / 영상 재생횟수 / 접속빈도 → "앱 쓰니까 회복 빨라졌다"
```

## ✏️ 발표에서 솔직하게 말할 것

- 7번 (다른 반려동물 입양)은 앱 범위 밖
- 생전 기록 (버킷리스트·일기)은 구현 진행 중
- 정량 분석은 데이터 수집 구조 구현 완료, 실데이터는 발표 시점 누적 예정
