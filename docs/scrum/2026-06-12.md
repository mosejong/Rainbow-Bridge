## 1. 주요 논의

### 1-1. ngrok → Cloudflare Tunnel 전환

- **이유:** ngrok 2시간 세션 만료로 LP/TTS 터널이 반복적으로 끊기는 문제 발생
- **결정:** LP(소람님)·TTS(환주님) 모두 `cloudflared tunnel --url` 방식으로 전환
- **결과:** LP Cloudflare 터널 정상 동작 확인. TTS 서버는 503 이슈로 계속 점검 중

### 1-2. Expo SDK 52 → 54 업그레이드

- **이유:** App Store 최신 Expo Go가 SDK 52 미지원 → iOS 실기기 테스트 불가
- **결정:** SDK 54로 업그레이드 완료 (민경이님 담당)
- **추후:** EAS Build + OTA 방식으로 빌드 예정 (iOS TestFlight 계정 공유 확인 후 확정)
- **강사 피드백:** iOS 실기기 테스트 확대, 3D 기능 정상 동작 여부 확인

### 1-3. 사진 자동 선택 로직

- **방식:** Gemini API로 해상도·정면 여부·선명도 등 점수 할당 후 자동 선택
- **변경:** media.jsx 수동 사진 선택 UI 제거 → 백엔드 `/api/v1/media/generate/{pet_id}` 자동 선택 엔드포인트 연결

### 1-4. LP GIF 524 타임아웃 해결

- **원인:** Cloudflare free tunnel 100초 하드컷 → GIF 생성(~77초) 중 연결 강제 종료
- **해결:** `server.py`에 비동기 job 엔드포인트 추가
  - `POST /generate/gif/async` — job_id 즉시 반환
  - `GET /generate/gif/status/{job_id}` — 상태 조회
  - `GET /generate/gif/result/{job_id}` — GIF 다운로드
  - 백엔드 `_generate_remote_gif` 30초 간격 폴링(최대 5분)으로 변경
- **추가 개선 (PR #250):** MP4 완료 후 GIF 순차 생성으로 변경 → 동시 GPU 세마포어 경쟁 해결

### 1-5. 삼성헬스 + 사용기록 LLM 연동 (정환주 → 반소람 핸드오프)

- 안드로이드 기준 삼성헬스 걸음·수면 데이터 + 앱 사용기록을 LLM에 전달
- 표준 출력 스키마: `{"steps": int|None, "sleep_hours": float|None}`
- 핸드오프 문서 공유 완료 (`HANDOFF_헬스데이터_스키마_반소람.md`)
- 상세 내용: 코드 위치 `ai/evaluation/health_export_adapter.py` · `health_signal.py`
- **강사 피드백:** 개인정보 비식별화 필요. 온디바이스 + 클라우드 하이브리드 구조 검토

### 1-6. TTS 안정화 방향 논의

- **현황:** Qwen3 TTS 서버 503/530 반복 → gTTS 폴백으로 운영 중. 음성 품질 저하
- **강사 피드백:**
  - API 기반 TTS(Qwen3 공식 API 등) 검토 — 환주님 담당
  - 감정형 프롬프트 테스트 진행
  - 타 팀 TTS 사례 참고

### 1-7. 서버 안정화 — 프로세스 모니터링 도입

- **현황:** uvicorn·cloudflared를 단순 프로세스로 실행 → 크래시 후 수동 재시작 반복
- **강사 피드백:** 프로세스 != 서비스. 자동 복구 체계 필요
- **결정:** 프로세스 감시 프로그램 직접 구현 검토 (모세종 공부 예정)
  - 핵심: heartbeat 체크 → 죽으면 자동 재시작 → 로그 기록
  - `psutil` + `subprocess` 기반 구현

### 1-8. 조기 통합 테스트

- **강사 피드백:** 기능 개발뿐 아니라 조기 통합 테스트 필요. 실기기 검증을 빠르게 진행할 것
- **우선순위:** TTS 안정화 → 서버 안정화 → 모바일 검증 → 기능 통합

### 1-9. 공모전

- 사회보장정보원 공모전 결과 발표 대기
- ICT 공모전 참가 여부 논의 중

## 2. 기술 이슈 해결

| 이슈 | 원인 | 해결 |
|------|------|------|
| 편지 낭독 TTS 무음 | `message.jsx`에서 audio_url 상대경로 그대로 사용 → `Audio.Sound` 조용히 실패 | `API_BASE` 절대경로 변환 추가 |
| TTS 504 Gateway Timeout | nginx 기본 60s < Qwen3 대기+폴백 처리 시간 | nginx `proxy_read_timeout 180s` 적용 |
| TTS gTTS 폴백 실패 | `except DefaultCredentialsError`만 잡아 ImportError 미처리 | `except (ImportError, DefaultCredentialsError)` 수정 |
| LP GIF 524 | Cloudflare 100초 하드컷 | 비동기 job 방식으로 전환 |
| 영상 재생 미동작 | `video_url` 상대경로 그대로 사용 | `url.startsWith('http')` 가드 후 `API_URL` 붙이기 |
| LP GIF GPU 세마포어 경쟁 | 업로드 시 GIF·MP4 동시 생성 → 10초 타임아웃 초과 시 error | MP4 완료 후 GIF 순차 생성으로 변경 (PR #250) |
| MongoDB DNS 해석 실패 | 컨테이너 재생성 시 이름 앞에 ID 붙음 → `rainbow_mongo` 이름 손실 | 전체 컨테이너 stop/rm 후 `up -d` 재생성 |
