# 레인보우 브릿지 — UI 흐름 설계

> 작성: 모세종 (PM) · 2026-06-10
> 대상: 민경이 (프론트 담당)
> 목적: 사용자 여정 기반 화면 흐름 정의 — **생존 모드 / 이별 후 모드** 분리

---

## 핵심 원칙

- 사용자는 **두 번 진입한다**: 반려동물이 살아있는 동안 / 이별 후
- 홈 화면은 **"오늘 뭘 해야 하는지"** 가 바로 보여야 함 (메뉴 나열 ❌)
- 이별 전환은 사용자가 **직접 누르는 버튼**으로 (강제 아님)
- 회복은 **점진적으로 열린다** — 위기 상태에서 추모 콘텐츠 강요 없음

---

## 전체 화면 맵

```
[온보딩] ─────────────────────────────────────────────┐
  스플래시 → 로그인 → 프로필 등록 → 추억 키워드 입력   │
                                                       ▼
                                          🌱 생존 모드 홈
                                                       │
                                          "무지개다리를 건넸어요" 버튼
                                                       │
                                                       ▼
                                          🌈 이별 후 모드 홈
```

## 1. 온보딩 (최초 1회)

> 사용자가 수의사에게 시한부 선고를 받고 앱을 처음 설치하는 순간

| 화면 | 주요 내용 | 비고 |
|------|---------|------|
| 스플래시 | "아이가 강아지별로 이사를 준비하고 있어요" | 온보딩 카피 |
| 로그인 | 이메일 + 비밀번호 | JWT 인증 |
| 프로필 등록 | 반려동물 이름·종·함께한 기간·보호자 호칭·사진 | 핵심 입력 |
| 추억 키워드 | 함께한 소중한 기억 최대 3개 | 추모 메시지 재료 |

**흐름:**
```
스플래시
  → 로그인 / 회원가입
  → 프로필 등록
  → 추억 키워드 입력
  → 🌱 생존 모드 홈
```

## 2. 🌱 생존 모드 (memorial_mode: false)

> "남은 시간을 함께 채우는" 단계 — 반려동물이 곁에 있는 동안

### 홈 화면

```
┌────────────────────────────────────┐
│  🌈 레인보우 브릿지                │
│  안녕하세요, [호칭]님 🐾           │
│  [이름]이와 함께한 오늘            │
│                                    │
│  ┌──────────────────────────────┐  │
│  │  📋 오늘 버킷리스트           │  │  ← 메인 기능 ①
│  │  □ 해 뜨는 거 같이 보기      │  │
│  │  □ 바다 보여주기             │  │
│  │  + 추가하기                  │  │
│  └──────────────────────────────┘  │
│                                    │
│  ┌──────────────────────────────┐  │
│  │  📔 오늘 일기 쓰기            │  │  ← 메인 기능 ②
│  │  "오늘 [이름]이와 있었던 일"  │  │
│  └──────────────────────────────┘  │
│                                    │
│  📷 사진 기록하기          →      │  ← 서브 기능
│  🏥 증상 안내 보기         →      │  ← 서브 기능
│                                    │
│  · · · · · · · · · · · · · · · ·  │
│  무지개다리를 건넜어요 🌈          │  ← 하단에 조용히
└────────────────────────────────────┘
```

### 메인 기능

| 기능 | 설명 |
|------|------|
| **버킷리스트** | 남은 시간 함께 하고 싶은 것 목록. 체크 완료 시 타임라인에 기록 |
| **일기** | 매일의 소중한 기억 기록. 날짜 선택·편집 가능 |

### 서브 기능

| 기능 | 설명 |
|------|------|
| 사진 기록 | 함께한 순간 사진 업로드 → 타임라인 연계 |
| 증상 안내 | 초기~말기 증상 변화 고정 텍스트 (의료 조언 대체 아님) |

### 네비게이션 (하단 탭)

```
[ 홈 ]  [ 기록 ]  [ 설정 ]
```
- **홈**: 버킷리스트 + 일기
- **기록**: 사진 타임라인
- **설정**: 프로필 수정

## 3. 이별 전환

> 보호자가 작별을 고한 뒤 직접 누르는 버튼

### 전환 버튼
- 위치: 생존 모드 홈 하단 — 눈에 잘 띄지 않게 (작게, 회색)
- 텍스트: `무지개다리를 건넜어요 🌈`

### 확인 모달
```
┌─────────────────────────────────┐
│                                 │
│  🌈                             │
│                                 │
│  [이름]이가 무지개다리를         │
│  건넜군요.                      │
│                                 │
│  곁에 있어드릴게요.             │
│                                 │
│  [ 네, 작별했어요 ]             │
│  [ 아직 함께 있어요 ]           │
│                                 │
└─────────────────────────────────┘
```
- "네" 선택 시: `PATCH /pets/{pet_id}` → `memorial_mode: true`
- 이별 후 모드 홈으로 전환

## 4. 🌈 이별 후 모드 (memorial_mode: true)

> 이별 후 감정을 돌보고, 천천히 일상을 되찾는 단계

### 홈 화면

```
┌────────────────────────────────────┐
│  🌈 레인보우 브릿지                │
│  [호칭]님, 오늘도 잘 버티고 있어요 │
│  [이름]이와 작별 후 D+3            │
│                                    │
│  ┌──────────────────────────────┐  │
│  │  오늘 감정은 어떠세요?        │  │  ← 메인 기능 ①
│  │  [ 체크인 하기 →  ]          │  │
│  └──────────────────────────────┘  │
│                                    │
│  ┌──────────────────────────────┐  │
│  │  🔒 [이름]이가 남긴 편지      │  │  ← 메인 기능 ②
│  │  회복도 ████████░░  63점     │  │    (회복 게이트)
│  │  80점이 되면 열립니다 🐾      │  │
│  └──────────────────────────────┘  │
│                                    │
│  ┌──────────────────────────────┐  │
│  │  ✅ 오늘의 미션               │  │  ← 메인 기능 ③
│  │  "집 앞 5분 산책하기"         │  │
│  │  [ 완료했어요 ]              │  │
│  └──────────────────────────────┘  │
└────────────────────────────────────┘
```

### 메인 기능

| 기능 | 설명 | 조건 |
|------|------|------|
| **감정 체크인** | 오늘 감정 점수 + 한 줄 기록. 위기 감지 자동 실행 | 매일 |
| **추모 편지** | AI가 추억 기반으로 생성한 편지 + TTS 낭독 + 영상 | 회복 게이트 통과 후 |
| **오늘의 미션** | 논문 기반 회복 미션 1개 추천 | 매일 |

### 편지 상태별 UI

| 회복도 | 상태 | 화면 |
|--------|------|------|
| ~49점 | 잠김 | 카드 없음 (너무 이른 시점) |
| 50~79점 | 잠김 + 예고 | 🔒 카드 노출 — 회복도 프로그레스바 |
| 80점+ | 해금 | ✉️ 편지 열기 가능 |
| risk_level 2+ | 강제 잠김 | 1393 안내 우선, 편지 차단 |

### 편지 해금 후 흐름

```
편지 열기
  → 텍스트 슬라이드 업 애니메이션 + BGM
  → TTS 낭독 (톤 선택: 따뜻하게 / 차분하게 / 부드럽게)
  → LivePortrait 추모 영상 (사진 있을 경우)
```

### 서브 기능

| 기능 | 설명 |
|------|------|
| 타임라인 | 생전 기록 + 감정 체크인 + 미션 완료 이력 |
| 리포트 | 감정 추이 꺾은선 그래프 + 회복 점수 + 미션 완료율 |

### 네비게이션 (하단 탭)

```
[ 홈 ]  [ 타임라인 ]  [ 리포트 ]  [ 설정 ]
```
