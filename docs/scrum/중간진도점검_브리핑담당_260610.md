### 3. AI 응답 엔진 (반소람)

| 순서 | 내용 | 강조 포인트 |
|------|------|------------|
| ① 엔진 선택 | Gemini 2.5-flash | 한국어 품질 + 무료 할당량 + OpenAI 호환 |
| ② 복원력 | 429 키 순환 → gemini-3-flash 폴백 → graceful 안내 | 강사님 6/9 피드백 전부 대응 |
| ③ 위기 감지 | L0(규칙) → L1(LLM JSON) → L2(위험) → L3(즉시 1393) | 골든셋 39건 미탐 0 |
| ④ RAG | corpus 63건 — memorial 45 / funeral 13 / mission 5 | category 필터로 컬렉션 분리 |
| ⑤ 1인칭 게이트 | `allow_first_person AND request_first_person` fail-closed | 경고 문구 + risk_level 0 조건 |
| ⑥ TTS 보이스 | boy / girl 2보이스 (Qwen3 로컬) | warm·calm·soft 톤 분기는 미구현 |

**예상 질문 대비**
- "comfort corpus가 0건인데 왜 컬렉션은 있는가?" → 현재 memorial로 커버, 다음 이터레이션 분리 예정
- "gemini-3-flash가 정식 모델명인가?" → Google AI Studio에서 최신 이름 확인 후 답변 준비
- "골든셋 39건 기준은?" → 팀이 직접 위기 시나리오 수동 작성, L2+ 케이스 포함

### 4. 백엔드 및 통합 (모세종 · 김윤한)

| 순서 | 내용 | 강조 포인트 |
|------|------|------------|
| ① 인프라 | NCP 실서버 (`rainbow-bridge.duckdns.org`) | 지금 켜져 있음 |
| ② 스택 | FastAPI + MongoDB + Redis | Redis = 체크인 7회 캐시, 반복 쿼리 절감 |
| ③ API 구조 | `api/` → `services/` → `models/` + `schemas/` | 레이어 엄수 |
| ④ 회복 게이트 | `GET /api/v1/emotions/recovery/{pet_id}` | 3단계 locked/teaser/open |

**예상 질문 대비**
- "실서버가 켜져 있는지 확인?" → 데모 직전 헬스체크 열어두기: `/health`
- "회복 게이트 점수 계산 로직?" → 최근 감정 체크인 기반 가중 평균, Redis → DB 순 조회
