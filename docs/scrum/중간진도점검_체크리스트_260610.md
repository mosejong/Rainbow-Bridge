## 1. 기획 및 페르소나

- [완료] 페르소나 한 줄 정의 확정
  - "시한부 선고를 받은 반려동물 보호자에게, 남은 시간을 함께 채우고 이별 후 일상을 되찾을 힘을 주는 AI 케어 서비스"

- [완료] 페르소나 카드 작성 (말투·가치관·지식 범위·금기 사항 포함)
  - 이름·세계관: 반려동물이 '강아지별로 이사'를 준비하는 동안 보호자 곁을 조용히 지키는 AI 케어
  - 말투·어조: 따뜻하고 차분한 존댓말. 감정을 서두르지 않음. 짧고 진심 있게.
  - 가치관·태도: 보호자의 속도에 맞춤. 회복을 강요하지 않음. 슬픔을 부끄럽지 않게 둠.
  - 전문성·지식 범위: 감정 공감, 추억 기록, 애도 지지, 장례 안내, 일상 복귀 미션
  - 모른다고 답할 영역: 의료 진단·처방, 수의사 역할 대체, 법률 판단
  - 금기 사항: 반려동물 부활·1인칭 대화(조건 미충족 시), 자해·자살 조장, 의료 조언 단정
  - 응답 형식: 짧고 위로가 되는 메시지. 위기(L2+) 시 1393 즉시 연결. 감정 강요 없음.
  - 첫 인사말: "아이가 강아지별로 이사를 준비하고 있어요. 남은 시간 동안 함께할게요."
  - 서비스 진입: 수의사가 직접 앱 추천 — B2B2C 연계형
  - 1인칭 허용 조건(강사 승인 2026-06-05): 보호자 동의 + 경고 문구 + risk_level 0 충족 시만

- [완료] 타깃 사용자 및 핵심 사용 시나리오 3개 정의
  - 타깃: 시한부 선고를 받은 반려동물 보호자 (펫로스 증후군 고위험군)
  1. 시한부 진입 — 수의사 선고 직후 보호자가 버킷리스트·추억 기록 시작
  2. 이별 직후 — 임종 후 "공백 시간", 장례 안내 + 위로 메시지로 감정 지지
  3. 회복 단계 — 감정 체크인 + 회복 게이트 통과 → 일상 복귀 미션·추모 편지 해금

- [완료] 기존 서비스 대비 차별점 명시
  - 단독 앱이 아닌 B2B2C 연계형 (병원·장례업체가 채널, 보호자 무료)
  - **펫로스 증후군 케어** 중심 — 시한부 선고부터 이별·회복까지 감정 돌봄 집중
  - 회복 게이트(`content_unlocked`) — 위기 상태 보호자에게 추모 콘텐츠 차단
  - 위기 감지 4단계(L0~L3) + 1393 자동 라우팅

- [완료] 결과물 형태 및 모달리티 선택 확정
  - 텍스트: Gemini 2.5-flash 기반 추모 메시지
  - 음성: Qwen3 로컬 TTS (추모 톤 진중함 최우수, VoiceDesign instruct 튜닝)
  - 영상: LivePortrait 사진→발화 영상 + FFmpeg faststart

## 3. AI 응답 엔진

- [완료] 응답 엔진 선정 완료 (근거 문서화)
  - **Gemini 2.5-flash** (OpenAI 호환 엔드포인트)
  - 선정 근거: 한국어 품질 우수 + 무료 할당량 + OpenAI SDK 호환으로 교체 용이
  - PERSO API 사용 안 함 (할당량 제한으로 메인 엔진 제외)

- [완료] 시스템 프롬프트로 페르소나 응답 동작 확인
  - `ai/llm/prompts/` 분리 관리
  - 가드레일 테스트 통과 (1인칭·부활 표현 미포함 검증)
  - 위기 감지 골든셋 39건 미탐 0 (L2+ 케이스 수동 작성·검증)

- [완료] 엔진 환경변수 스위칭 구조 확보
  - `.env` — `LLM_PROVIDER` / `LLM_BASE_URL` / `LLM_MODEL` / `LLM_API_KEY` 분리
  - 값만 바꾸면 Gemini ↔ Ollama ↔ PERSO 전환 가능
  - **LLM 복원력 3건 (강사님 6/9 피드백 대응 완료, PR #150)**
    1. 429(할당량 소진) → 다음 키로 즉시 전환 (`LLM_API_KEY=key1,key2` 쉼표 분리)
    2. 전 키 소진 → 폴백 모델(`LLM_FALLBACK_MODEL=gemini-3-flash`)로 재시도
    3. 타임아웃·5xx → 지수 백오프 재시도, 최종 실패 시 `LLMError` → graceful 안내문 반환 (앱 500 미발생)

- [완료] 파이프라인
  ```
  사용자 → React Native → FastAPI
    ├─ memorial  → RAG(ChromaDB) + Gemini + 가드레일 → 추모 메시지
    ├─ mission   → LLM + 규칙 폴백 → 일상 복귀 미션
    ├─ safety    → 규칙(L0) + LLM JSON 분류(L1) → 4등급 판정 → 1393 라우팅
    └─ funeral / terminal_care → RAG(funeral 컬렉션) → 안내 응답
  ```

## 6. 평가 설계

- [완료] 평가 지표 3~5개 선정 및 가중치 설정

  | 지표 | 가중치 | 측정 방법 |
  |------|--------|-----------|
  | 위기 감지 미탐률 | 40% | 골든셋 39건 기준 L2+ 미탐 0 |
  | 메시지 가드레일 통과율 | 30% | 1인칭·부활 표현 미포함 비율 |
  | 회복 게이트 정확도 | 20% | 단위 테스트 8종 통과 |
  | TTS 보이스 품질 | 10% | boy/girl 2보이스 추모 낭독 청취 검증 |

  **가중치 선정 근거 및 이론적 배경**

  | 지표 | 근거 |
  |------|------|
  | 위기 감지 미탐률 (40%) | 정신건강 AI에서 **Recall 우선 원칙** — 미탐(놓침)이 오탐보다 치명적. 펫로스는 복잡성 애도(Complicated Grief, Shear 2015)로 이어질 위험이 있어 조기 감지가 핵심 |
  | 메시지 가드레일 통과율 (30%) | **Continuing Bonds 이론** (Klass, Silverman & Nickman, 1996) — 고인과의 관계는 끊어지는 것이 아니라 재정의됨. 이 맥락에서 AI가 "부활" 또는 "실제처럼 대화"하는 건 이론적으로도 부적절 |
  | 회복 게이트 정확도 (20%) | **애도 작업 4과제 모델** (Worden, 2009) — 상실 수용 → 감정 처리 → 적응 → 관계 재정의 순서. 게이트는 이 단계를 건너뛰지 않도록 설계 |
  | TTS 보이스 품질 (10%) | 감정적 내러티브에서 음성 톤이 감정 이입에 영향 (Nass & Brave, 2005 "Wired for Speech") |

- [완료] 카테고리 균형 테스트셋 작성
  - corpus.json 63건: memorial 45 / funeral 13 / mission 5 / comfort 0
  - 위기 감지 골든셋 39건 (L0~L3 등급별 수동 작성)

- [미착수] 평가 지표별 자동화 채점 스크립트
  - 완료 계획: 골든셋 기준 pytest 기반 자동 평가 추가
