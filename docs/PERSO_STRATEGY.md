# PERSO AI 활용 전략

> **최종 수정:** 2026-06-05
> **담당:** 장민수 (멀티모달) · 모세종 (PM)

---

## 1. PERSO를 왜 이 위치에 배치했나

PERSO AI는 **영상 더빙·번역·STT** 전용 API입니다.
텍스트 생성(LLM)과 독립 TTS 기능은 없음이 공식 확인됐습니다.

| 기능 | PERSO 가능? |
|------|------------|
| 텍스트 생성 (LLM) | ❌ 엔드포인트 없음 |
| 단독 TTS | ❌ 더빙 프로젝트 내부 전용 |
| 영상 더빙·번역 | ✅ |
| 립싱크 (영상 입력) | ✅ |
| STT (자막 추출) | ✅ |

따라서 PERSO를 **추모 메시지 생성이나 낭독에 쓰지 않고**,
완성된 추모 영상을 후처리·패키징하는 역할에 배치했습니다.

---

## 2. 서비스 내 역할 분담

```
Gemini API        → 추모 편지 생성 / 미션 추천 / 위기감지 보조
Google Cloud TTS  → 보호자 대상 편지 낭독 (warm / calm / soft 톤)
LivePortrait      → 반려동물 사진 기반 잔잔한 추모 영상 생성
PERSO AI          → 완성 영상 후처리 (자막·더빙·다운로드 산출물 패키징)
```

---

## 3. 윤리 경계

PERSO 활용 시 **절대 금지:**

- 반려동물 입모양을 맞추는 립싱크 (`lipSync=false` 필수)
- 반려동물이 직접 말하는 것처럼 보이는 영상 생성
- "나는 잘 지내요", "엄마 울지 마" 등 반려동물 1인칭 발화

PERSO는 보호자가 **간직하거나 가족과 공유**할 수 있는
**추모 산출물 완성**에만 사용합니다.

---

## 4. 구현 구조 (비동기)

발표 시연에서 PERSO 실시간 호출은 하지 않습니다.
외부 API 처리 시간(업로드·작업 대기)이 발생하기 때문에
**비동기 작업 큐 구조**로 설계합니다.

```
POST /api/v1/media/{asset_id}/perso   # 후처리 작업 요청
GET  /api/v1/media/{asset_id}/perso/status  # 작업 상태 조회

상태값: pending → processing → completed / failed
```

시연에서는 사전 생성된 샘플로 `completed` 상태를 보여줍니다.

---

## 5. 시연 시나리오

```
1. 보호자가 반려동물 사진 업로드
2. LivePortrait → 잔잔한 추모 영상 생성 (ambient 움직임)
3. Google TTS → 추모 편지 낭독 mp3 생성
4. FFmpeg → 영상 + 오디오 합치기
5. "PERSO 후처리 요청" 버튼 클릭
   → 상태: processing
6. 완성된 샘플 결과 → completed 표시
7. MP4 다운로드 / 재생
```

---

## 6. 발표 멘트

> PERSO AI는 반려동물을 말하게 만드는 용도가 아니라,
> 보호자가 남긴 추모 콘텐츠를 보관 가능한 영상 산출물로
> 완성하는 데 사용했습니다.
>
> Gemini는 기억 기반 추모 편지와 회복 미션을 담당하고,
> Google TTS는 보호자 대상 낭독 음성을 생성합니다.
> LivePortrait는 사진을 기반으로 잔잔한 추모 영상을 만들며,
> PERSO AI는 이 산출물을 다운로드 가능한 형태로 패키징합니다.
>
> PERSO를 범용 AI로 쓰지 않고,
> 영상·오디오 처리에 강한 API 성격에 맞춰 배치했습니다.

---

## 7. 향후 확장 가능성

- 추모 영상 다국어 자막/더빙 → 해외 가족과 공유
- STT로 보호자 육성 녹음 → 타임라인 음성 기록
