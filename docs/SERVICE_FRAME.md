## 1. 핵심 재정의 — "단독 앱" → "연계형 서비스"

펫로스는 **일회성·특수 상황**이라 그것만으로 앱을 설치·유지하기 어렵다.
→ 우리 앱 하나를 깔게 하지 말고, **사람·데이터가 이미 있는 접점에 얹는다.**

- ❌ "보호자가 펫로스 때문에 우리 앱을 검색·설치"
- ✅ "이미 거치는 접점(장례·병원·펫앱)에서 **자연스럽게 우리 기능으로 넘어옴**"

구조: **B2B2C**(장례·병원·펫앱이 우리를 탑재 → 그들 고객에게 제공) + **API 연계**.

### MVP 쐐기 — 병원→장례 "공백" ⭐

임종 직후, 보호자가 멘붕 상태로 혼자 나오는 **그 몇 시간~며칠**을 아무 서비스도 안 채운다.
→ "병원·장례 두 곳에 등장"이 아니라, **그 사이의 빈 곳(임종→장례 전환)** 을 메우는 게 1차 쐐기.

- **감정 진입 타이밍 최강** — 가장 도움이 절실한 순간
- **데이터 자연 유입** — 병원·장례 접점에서 반려동물 정보·기록이 넘어옴(보호자가 백지 입력 안 함)
- **검증된 흐름** — 임종→장례는 반드시 일어나는 행동 ("그림이 안 그려진다" 해소)
- 코어 기능 [funeral.py](../ai/llm/funeral.py)(장례 절차 상담) **이미 구현**

### 4자 역할 (김윤한 정리 기반)

| 주체 | 역할 |
|------|------|
| **병원·장례지도사** | 보호자에게 앱 **연결(채널)** + B2B **지불자** |
| **보호자** | 앱으로 **회복** (무료) |
| **수의사** | 회복 과정 **모니터링·조언** |
| **우리** | 연결·운영·**수익화** |

> 🌈 태그라인: **"레인보우 브릿지 — 병원에서도, 장례식장에서도 만날 수 있습니다."**

## 2. 진입점 — 다중 진입(연계 파트너)

> 단일 진입을 고집하지 않는다. 반려동물 생애주기 단계마다 다른 파트너가 우리를 부른다.

| 진입 파트너 | 시점 | 역할 | 비고 |
|-------------|------|------|------|
| **동물병원/수의사** | 아플 때~임종 | 건강기록 ↔ 회복 리포트, 사후관리 연계 | 기존 [SERVICE_DIRECTION](SERVICE_DIRECTION.md) 축 |
| **장례지도사/펫 장례업체** | 임종·장례 | **장례 시점 진입** → 추모·미션 시작 | 🔵 **강사 제안(2026-06-08)** — 데이터 수집·검증 해결책 |
| **반려동물 종합 케어앱** | 생애주기 전체 | 우리는 그 **"추모 모듈"** 로 탑재 | 프로필·사진 그대로 이어받음 |
| **심리상담 플랫폼** | 위기·회복단계 | 전문 상담 **핸드오프** | `welfare_resources` 일부 설계됨 |

> ✅ **1차 진입점 결정(2026-06-08): 병원→장례 "공백"** — 임종→장례 전환 순간을 메우는 것을 MVP 쐐기로. 병원·장례지도사 둘 다 채널이고, **그 사이를 우리가 연결**. (수의사 모니터링·펫앱 연계는 확장 단계)

## 💰 수익모델 — 누가 돈을 내나

> 🚨 **보호자(슬픔 중)에게 직접 팔지 않는다.** 무료로 잡고, 수익은 생태계(B2B)에서.

| 지불자 | 무엇에 | 왜 낸다 (그들의 이득) |
|--------|--------|------------------------|
| **장례업체** ⭐핵심 | **송객 수수료** (임종 직후 검증된 업체로 고객 연결) | **전환율 최고 타이밍**의 리드. 장례비 평균 46만원 → 케이스당 실속 |
| **동물병원** | SaaS 구독·제휴 | "사후관리하는 병원" 차별화·재방문(다음 반려동물)·안락사 후 케어 부담↓ |
| **보호자(B2C)** | 가벼운 프리미엄만 | 첫 추모 편지·영상 **무료**(회복 보상), 추가 생성·고화질·구독만 유료 |
| **부가** | 커미션 | 메모리얼 굿즈(유골함·액자)·펫보험·커머스 제휴 |

→ **주 수익 = B2B**(장례 송객 + 병원 제휴), 보호자 과금은 보조.
> 한 줄: **"보호자에겐 공짜 위로, 수익은 그들이 어차피 거치는 생태계에서."** = 윤리(슬픔에 과금 ❌) + 사업성 동시.

## 3. 기능 흐름 (강사 제시 — 채택)

```
[장례]  →  [일상 미션]  →  [복귀]  →  [추모·기념]  →  [건강관리·병원 연계]
 진입       회복 유도       일상화      보고싶을 때        (다음 반려동물/지속 관계)
```

- **장례**: 장례지도사/업체 연계로 진입. 반려동물 정보·장례 일정 수집.
- **일상 미션**: 회복 단계별 행동 미션(밖으로 향하는 일상 복귀). [RECOVERY_GATE](RECOVERY_GATE.md)
- **복귀**: 회복 점수 추적, 무너지면 돌봄(1393)으로 전환.
- **추모·기념**: 회복할수록 표현이 깊어짐(추모 메시지·영상). 기념일 케어(D+30·D+100).
- **건강관리·병원 연계**: 다음 반려동물/지속 관계 → 병원 연계로 순환.
