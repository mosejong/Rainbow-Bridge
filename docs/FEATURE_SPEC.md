# 기능 정의서 — 레인보우 브릿지 (Rainbow Bridge)

> **작성:** 2026-06-08 · 모세종  
> **스크럼 공유용**

---

## 서비스 한 줄 정의

> **"반려동물을 잃은 보호자의 펫로스 증후군 회복을 수의사와 연계해 돕는 AI 케어 서비스"**

---

## 사용자 역할

| 역할 | 설명 | 진입 경로 |
|------|------|-----------|
| **보호자** | 반려동물을 잃고 펫로스를 겪는 사람 | 모바일 앱 |
| **수의사** | 보호자 기록 확인 + 회복 조언 제공 | 수의사 전용 웹 |

---

## 핵심 기능 목록

### 보호자 앱 (8개 프로토타입)

| # | 기능명 | 정의 | 주요 API | 상태 |
|---|--------|------|----------|------|
| ① | **반려동물 프로필 입력** | 이름·종·함께한 기간·추억(keyword+detail) 등록 | `POST /pets` | ✅ |
| ② | **감정 체크인** | 1~10점 감정 점수 + 메모 입력. 위기 감지(risk_level 0~3) 자동 판별 | `POST /emotions` | ✅ |
| ③ | **AI 추모 메시지 생성** | 보호자 기억·감정 기반 위로 메시지. RAG·가드레일·위기 감지 통합 적용 | `POST /messages` | ✅ |
| ④ | **TTS 낭독** | 메시지를 음성으로 변환. warm·calm·hopeful 3가지 톤 선택 | `POST /tts` | 🟡 엔진 확정 중 |
| ⑤ | **일상 복귀 미션 추천** | 회복 단계별 행동 미션 AI 추천 (LLM+RAG, 풀 30개) | `GET /missions/{pet_id}` | ✅ |
| ⑥ | **추모 타임라인** | 감정·메시지·미션 완료 등 기록을 시간순 저장·조회 | `GET /timeline/{pet_id}` | ✅ |
| ⑦ | **위기 감정 안전 라우팅** | risk_level 2 이상 → 자살예방상담전화 1393 자동 안내. L0 규칙+L1 LLM 다층 감지 | 감정·메시지 API 내 통합 | ✅ |
| ⑧ | **회복 평가 리포트** | 감정 추이·미션 완료율·위기 횟수 집계. 수의사 조회 가능 | `GET /report/{pet_id}` | 🟡 실연결 중 |

### 단계별 케어 모듈 (AI — 펫로스 여정 1~3단계)

> ③⑤⑦ 외에 반소람이 구현한 추가 LLM 모듈. 모두 보호자 위기 선체크(1393) 내장.

| 단계 | 모듈 | 정의 | 상태 |
|------|------|------|------|
| 1단계 | **증상 진료 안내** | 반려동물 증상 → 진료 안내 생성 + 보호자 위기 선체크 (`triage.py`) | ✅ 골격 / vet_protocol RAG 미연결 |
| 2단계 | **장례 절차 단계별 상담** | 장례 단계별 안내 + 질문 RAG 응답 (`funeral.py`, funeral RAG) | ✅ |
| 3단계 | **기념일 케어 (D+30·D+100)** | 무지개다리 기념일 감지 → 보호자 케어 메시지 (`anniversary.py`) | ✅ |

### 가산점 기능 — 멀티모달

| 기능명 | 정의 | 상태 |
|--------|------|------|
| **사진 → 추모 영상** | 반려동물 사진을 LivePortrait로 자연스럽게 움직이는 영상 생성 | ✅ (로컬) / 🟡 원격 GPU |
| **말하는 동물 영상** | 발화 driving 영상으로 사진 속 반려동물이 "말하는 것처럼" 생성 (입 다문 사진 권장) | ✅ 검증 (Day 6) |
| **다양한 동물 종 지원** | 강아지·고양이 외 토끼·햄스터·앵무새·올빼미·말 등 비전형 7종 검출·생성 | ✅ 검증 (Day 4) |
| **영상 + TTS 합성** | 추모 영상에 TTS 음성을 FFmpeg로 합쳐 최종 파일 제공 | ✅ |
| **다운로드 제공** | 생성된 영상·음성 파일 다운로드 | ✅ |

## 핵심 설계 원칙

| 원칙 | 내용 |
|------|------|
| **안전 최우선** | risk_level 2+ → 1393 즉시 안내. 생성 중단. |
| **반려동물 부활 금지** | AI는 보호자를 위로하는 역할. 반려동물인 척 말하지 않음 |
| **1인칭 조건부 허용** | 보호자 동의 + 경고 문구 + risk_level 0~1 동시 충족 시만 |
| **수의사 연계** | 병원에서 앱 추천 → 수의사가 보호자 상태 모니터링 |
| **의학적 근거 미션** | 펫로스 치료 논문(Continuing Bonds 등) 기반 행동 미션 (적용 예정) |
| **멀티모달 윤리·안전** | 말하는/1인칭 영상엔 "AI 재해석" 라벨 필수. 위기 상황(risk_level 2+) 시 영상 생성 금지 |
| **전문 치료는 사람이** | CBT 등 실제 심리치료는 AI가 아니라 상담전문가·정신건강의학과 전문의가 담당. 앱은 감정 돌봄·위기 감지 후 전문가·복지자원으로 **연계**만 (의료·심리치료 대체 아님) |

## 향후 계획 (검증·설계만, 구현은 회의 후)

| 항목 | 내용 |
|------|------|
| **driving 자동 선택** | 입력 사진 입 개폐를 XPose로 판정해 최적 driving 자동 매칭 (검증 완료) |
| **사진 입력 가이드** | 입 다문 정면 사진 권장 — 발화 영상 품질 확보 |

## 회의 안건

- 튜터님 *"사진 넣으면 그 아이가 말하는 영상 = 추억의 핵심"* 방향이면, **"말하는 동물 영상"을 가산점이 아니라 핵심 기능으로 승격**할지 회의 때 결정.
