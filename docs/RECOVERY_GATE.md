# 회복 게이트 설계 — Recovery Gate

> **작성:** 2026-06-07 · 모세종
> **최종 수정:** 2026-06-08 — 1회 언락·사후 모니터링·수익 모델 반영
> **목적:** 보호자의 회복 상태에 따라 콘텐츠를 단계별로 제공 — 초기 취약 시점에 강한 자극(펫 1인칭 편지·영상)을 막고, 회복 동기를 부여하는 구조

---

## 왜 이게 필요한가

펫로스 초기에는 반려동물의 목소리·편지를 바로 보여주면 **집착·현실 부정**으로 이어질 수 있다.
회복이 어느 정도 된 시점에 제공해야 치유 효과가 생긴다.

→ 회복 점수를 추적하고, 점수에 따라 콘텐츠를 단계적으로 언락한다.
→ "회복하면 OO이의 편지를 받을 수 있다"는 **찌라시**가 회복 동기를 만든다.

---

## 회복 점수 (0~100)

### 산식

```
recovery_score =
    감정 평균(40%) + 미션 완료율(35%) + 체크인 꾸준함(25%)
```

| 요소 | 계산 방법 | 가중치 |
|------|-----------|--------|
| 감정 평균 | 최근 7회 체크인 평균 (1~5점 → 0~100 변환) | 40% |
| 미션 완료율 | 완료 미션 수 / 추천 미션 수 × 100 | 35% |
| 체크인 꾸준함 | 최근 14일 중 체크인한 날 수 / 14 × 100 | 25% |

### 안전 게이트

- **risk_level 2 이상이면 점수 계산 중단** — 1393 안내 우선, 언락 없음
- risk_level이 다시 0~1로 내려오면 점수 재개
- 점수는 낮아질 수도 있다 — 억지로 올리는 구조 아님

### 초기값

- 체크인 기록 없을 때: 50점 (중립 시작)
- 미션 추천 전: 미션 완료율 0% 아닌 null 처리 → 감정 평균만으로 계산

---

## 콘텐츠 단계

| 점수 | 상태 | 제공 콘텐츠 |
|------|------|------------|
| 0 ~ 49 | 초기·위기 | 위로 메시지 (보호자 입장), 미션 추천, 타임라인 |
| 50 ~ 79 | 회복 중 | 위 전체 + **찌라시 카드** (잠김 상태로 노출) |
| 80+ | 회복 | **기억 기반 추모 편지 최초 1회 무료 생성** + LivePortrait 추모 영상 |

> 80점 기준은 추후 실사용 피드백 보고 조정 가능.

---

## 언락 구조 — 1회 영구 언락

> ⚠️ **반복 게이팅 없음.** 80점 도달 시 편지를 **1회 무료로 생성**하고 영구 저장.
> 이후 점수가 떨어져도 기존 편지는 계속 볼 수 있음 (DB에서 조회, 토큰 소비 없음).

| 상황 | 동작 |
|------|------|
| 최초 80점 달성 | 편지 생성 (무료, 토큰 1회 소비) → DB 저장 → 영구 열람 가능 |
| 점수 하락 후 재상승 | 기존 편지 조회만 가능 (토큰 없음) |
| 새 편지 재생성 요청 | **유료 크레딧 차감** |

**왜 이 구조인가:**
- 악용 방지: 점수 올렸다 내렸다 반복해도 편지를 새로 받을 수 없음
- 토큰 비용 관리: 생성은 1회, 이후 조회는 무료
- 수익 모델: 새 편지를 원하면 크레딧 구매

---

## 수익 모델 — 크레딧

| 항목 | 가격 | 비고 |
|------|------|------|
| 첫 번째 편지 | **무료** | 80점 달성 보상 |
| 추가 편지 생성 | ₩2,900 / 1회 | 크레딧 단건 구매 |
| 월 구독 (예시) | ₩4,900 / 월 | 편지 2회 + 영상 포함 |

> 구체적인 가격은 팀 협의 후 결정. 발표 데모에서는 크레딧 시스템 UI만 보여줘도 됨.

---

## 편지 열람 후 모니터링

> 편지 수신 후 감정이 오히려 떨어질 수 있다는 피드백 반영.

**열람 전 경고 문구:**
> "이 편지는 감정적으로 강한 내용을 담고 있을 수 있어요. 충분히 준비됐을 때 열어보세요."

**열람 후 7일 집중 추적:**
- 체크인 점수가 -20점 이상 하락하면 → 위로 메시지 강화 + 미션 강도 낮춤
- risk_level 2+ 진입 시 → 1393 안내 즉시 발동

```
편지 열람
    │
    ▼
7일 모니터링 시작
    │
    ├─ 점수 유지/상승 → 정상 흐름
    └─ 점수 -20 이상 하락 → 위로 메시지 강화 + 미션 재조정
                           └─ risk_level 2+ → 1393 즉시
```

## 찌라시 카드 (50점부터 노출)

```
┌─────────────────────────────────────────┐
│  🔒  OO이가 남긴 편지                    │
│                                         │
│  OO이와의 추억을 바탕으로 쓴              │
│  특별한 편지가 기다리고 있어요.           │
│                                         │
│  회복도  ██████████░░░░  63점           │
│          80점이 되면 열립니다            │
│                                         │
│  천천히 괜찮아요 🐾                      │
└─────────────────────────────────────────┘
```

**문구 원칙:**
- 숫자보다 감정 언어 우선: "조금만 더 힘내면 OO이가 기다리고 있어요"
- 압박감 주지 않기: "천천히 괜찮아요" 부문구 필수
- 언락 후 면책 문구 필수: "AI가 보호자가 전해준 추억을 바탕으로 재해석한 편지입니다"

## API 설계

### `GET /api/v1/pets/{pet_id}/recovery`

```json
{
  "pet_id": "...",
  "recovery_score": 63,
  "status": "recovering",
  "unlocked": {
    "memorial_letter": false,
    "liveportrait_video": false
  },
  "letter_generated": false,        // 최초 생성 여부 (true면 DB에서 조회)
  "letter_credits": 0,              // 보유 크레딧 (추가 생성용)
  "next_milestone": {
    "score": 80,
    "reward": "OO이의 추억 편지 (무료)"
  },
  "risk_gated": false,
  "post_letter_monitoring": false   // 편지 열람 후 7일 모니터링 중 여부
}
```

### 점수 갱신 트리거

- `POST /emotions` (감정 체크인) → 즉시 재계산
- `PATCH /missions/{id}/complete` (미션 완료) → 즉시 재계산
- 재계산 결과 Redis에 캐시 (`recovery:{pet_id}`, TTL 1시간)

---

## 기존 아키텍처 연결

```
감정 체크인 ②  ──┐
미션 완료 ⑤    ──┼──▶ recovery.calculate(pet_id) ──▶ Redis 캐시
체크인 이력     ──┘         │
                            ▼
                  GET /pets/{id}/recovery
                            │
              ┌─────────────┴──────────────┐
           score < 80                  score >= 80 (최초)
          찌라시 카드 노출           편지 1회 무료 생성 → 영구 저장
                                          │
                                    이후 재요청
                                          │
                              크레딧 있음? → 재생성
                              크레딧 없음? → 구매 유도
```

## 윤리 고려사항

- 점수가 낮아도 **강제하지 않는다** — 보호자 페이스 존중
- 80점이 "완치"를 의미하지 않는다 — UI에서 "충분히 준비됐어요" 표현 권장
- risk_level 2+ 상태에서 언락 없음 — 취약 시점 강한 자극 차단
- 언락 후 면책 문구 항상 표시 — "AI가 재해석한 편지" 명시
- 1인칭 화법은 80점+ + risk_level 0~1 **동시 충족** 시에만 허용 (강사 승인 기준)
- 편지 열람 후 7일 모니터링 — 재우울 조기 감지
