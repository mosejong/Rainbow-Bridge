## 0. 핵심 — 가중치 발명하지 않는다

팀은 이미 [RECOVERY_GATE.md §회복 점수](../../docs/RECOVERY_GATE.md)에 **감정 40% / 미션완료율 35% / 꾸준함 25%** 를 합의해 뒀다.
→ 새 가중치를 만들지 않고 **이 설계를 채택**한다. 논문(효과크기·engagement)은 이 설계의 **"방법론 근거"** 로만 붙인다.
→ 효과크기로 카테고리/요소를 **차등 가중하지 않는다**(이종 모집단 숫자 = false precision = 과장 금지 위반).

## 1. 점수가 2개로 갈린다 (혼동 주의)

| 점수 | 정체 | 공식 | 파일 | 담당 | 현재 상태 |
|------|------|------|------|------|----------|
| **게이트 점수** `recovery_score` | 콘텐츠 언락(0~100, 80점=편지 1회 무료) | 40/35/25 | `services/recovery.py` (지금은 `emotion.py` 임시) | **모세종** | `round(avg*10)` 한 줄 (미구현) |
| **일상복귀 신호** `recovery_signal` | 발표·리포트 정량(회복중/유지/주의 + 추세) | **40/35/25** | [recovery_signal.py](recovery_signal.py) | **정환주 ⑧** | recovery_index=40/35/25 + 꾸준함 + `recovery_score` 순수함수 (리뷰 반영) |

→ 둘은 **다른 물건.** ⑧ 신호는 **구현 끝**. 게이트 점수(콘텐츠 언락)는 모세종이 `recovery_score` 순수함수를 import해 risk게이트·초기 50만 감싸면 됨.

## 2. 3-way 비교분석 (현재 코드 / 설계 / 논문)

| 요소 | 현재 코드 | RECOVERY_GATE 설계 | 논문 근거 |
|------|----------|-------------------|----------|
| 점수 범위 | `round(avg*10)` → 10~100 | 0~100 | ICG 0~76 (>25=기능 손상), α=0.93 (Prigerson 1995, PubMed 8771222) |
| 감정 | **100%** | **40%** | 주관적 비탄 자기보고 (ICG 계열) |
| 미션 완료율 | 없음 | **35%** (완료/추천×100) | 행동활성화 d=0.8~0.9 ⚠ · 표현적 글쓰기 g=0.388 |
| 꾸준함(14일) | 없음 | **25%** (체크인 날수/14×100) | engagement/adherence = 행동마커 (PMC11303913, 다운로드자 65% 1주 내 이탈) |
| 초기값 | **0점** | **50점** (중립) | — |
| risk≥2 | 창 내 최대 risk로 잠금 | **계산 중단·언락 없음**, 1393 우선 | 외부화 유대 차단 (Continuing Bonds) |

→ **격차:** 현재는 감정 100%·초기 0점. 설계(40/35/25·초기 50)는 **미구현**.

## 3. 점수 설계 초안 (산식)

### 3-A. 게이트 점수 `recovery_score` (0~100)

```text
if risk_level >= 2:
    return None            # 1393 우선, 언락 없음 (RECOVERY_GATE §안전 게이트) — 게이트 호출부 책임

E = clamp((최근7회_감정평균 - 1) / 9 * 100, 0, 100)   # 감정 1~10 → 0~100, 범위 클램프
M = 완료미션 / 추천미션 * 100                           # 추천 0이면 M = null
C = 최근14일_체크인한_날수 / 14 * 100                    # 꾸준함

if 체크인 기록 없음:   recovery_score = 50                # 중립 시작 (게이트 호출부 책임)
elif M is null:        recovery_score = round(E)          # 미션 전: 감정 평균만 (설계 그대로)
else:                  recovery_score = round(0.40*E + 0.35*M + 0.25*C)
# 최종 0~100 클램프 보장.
```

#### 콘텐츠 단계 (RECOVERY_GATE §콘텐츠 단계 그대로)
| 점수 | 상태 | 콘텐츠 |
|------|------|--------|
| 0~49 | 초기·위기 | 위로 메시지·미션·타임라인 |
| 50~79 | 회복 중 | + 찌라시 카드(잠김 노출) |
| 80+ | 회복 | 편지 1회 무료 + 추모 영상 (risk 0~1 동시 충족 시 1인칭 허용) |

### 3-B. 내가 박은 판단 2건 (← 여기부터 분석)

1. **감정 정규화 `(avg-1)/9*100` 채택**
   - 현재 `avg*10`은 최저가 10점이라 "바닥 0·초기 50" 설계와 안 맞음.
   - `(avg-1)/9*100`이면 1점→0, 10점→100, 중립 5.5점→50 근처로 설계 정합.
   - (대안: 설계 문구 "1~5점→0~100"은 5점 척도 가정인데, 실제 스키마는 **1~10** → 1~10 기준 정규화가 맞음.)

2. **미션 추천 전(M=null) = 감정 평균만** (리뷰 후 수정)
   - 처음엔 0.615E+0.385C로 재분배했으나, 리뷰가 "설계엔 없는 임의 비율"이라 지적.
   - RECOVERY_GATE §초기값 문구 그대로 **"감정 평균만으로 계산"** = `recovery_score = round(E)` 로 변경.
   - ⚠️ 부작용: None(미추천)이 0.0(미션 전부 실패)보다 점수 높음 = **의도된 순서**(데이터 없음=중립 vs 낮은 참여). 사용자가 미션 추천 여부를 못 고르므로 악용 불가 — docstring에 명시.

### 3-C. ⑧ `recovery_signal` 정합 — 구현 + 리뷰 반영
- 신호 라벨(회복중/유지/주의)·빈도 추세(접속/재생) = **현행 유지**.
- `recovery_index` `avg*10` → **3-A `recovery_score` 산식으로 교체** (리포트 숫자 = 게이트 숫자).
- 감정 평균 = **최근 7회**(`_RECENT_WINDOW`, 리뷰 후 수정). 추이(older/recent)는 전체 사용.
- **꾸준함(C) 추가** — `_consistency()`가 체크인 `created_at`에서 14일 체크인 날수 산정.
- 출력에 `checkin_consistency` 키 + "체크인 꾸준함 N%" 근거 문장 추가.
- **`recovery_score(...)` 순수함수** 신설(클램프 0~100 보장) → 모세종 게이트가 그대로 import 가능.
- 검증: `test_recovery_signal.py` **17/17** (가중합·감정only·클램프·7회윈도우·index정합·꾸준함).
