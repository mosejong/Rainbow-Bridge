## 1. 왜

지금 회복점수(`recovery_signal.py`)는 **주관 감정 체크인(1~10)** 위주 → 번거롭고 주관적.
삼성헬스에 이미 쌓이는 **수면·활동**을 더해 ① 객관성 보강, ② "센서는 나쁜데 본인은 괜찮다더라" 불일치 포착.

## 2. 데이터 경로 (검증 2026-06-11)

```
삼성헬스 → (설정에서 Health Connect 동기화 ON) → Health Connect → react-native-health-connect@3.5.3 → 앱
```
- **Expo Go ❌ / 개발 빌드 ⭕** (네이티브 모듈). 삼성헬스 ≠ Health Connect(별개 생태계, 동기화 ON 필수).
- 안드로이드 전용. 팀 Expo Go 워크플로 안 건드림 — 헬스 모듈은 공유 앱에 안 넣고 정환주 개발빌드 브랜치에 격리.

## 3. 산식 (핵심 40/35/25 + 활동 add-on)

| 항목 | 가중치 | 정규화 | 점수 반영 |
|------|--------|--------|-----------|
| 감정(주관) | 40 | `(avg-1)/9*100` | ✅ 핵심(항상) |
| 미션 누적 | 35 | 1미션=1점, **천장 35** | ✅ 핵심(항상) |
| 체크인 꾸준함 | 25 | 최근14일 체크인날/14 | ✅ 핵심(항상) |
| **활동(객관)** | **10** | 걸음/8000*100 | ⭕ 들어올 때만(옵션 A) |
| ~~수면(객관)~~ | ~~제외~~ | 삼성헬스 수면점수/시간 | ❌ **점수 제외** → 교차검증·표시·미션난이도만 |

> ⚠️ 가중치는 **설계 판단**(`health_signal.py` 맨 위 `W_*`)이나, **40/35/25는 결정문서 §1
> 정본**·**수면 제외는 §2 결정**이라 임의 변경 금지.
> 기존 `recovery_score()`(40/35/25)는 **안 건드림** — 활동 미제공 시 그대로(하위호환).
> 미션 캡은 base·blend **둘 다 35**로 통일 — 달랐다간 base→blend 전환 시 활동과 무관하게
> 점수가 튐(reviewer 2026-06-11 발견, `_MISSION_CAP=35`).
>
> **부분 데이터 정책 = A(재정규화):** 빠진 외부신호(활동)는 **분모에서 제외** →
> 데이터 없다고 점수 안 깎임. 핵심항(감정·미션·꾸준)은 항상 분모 포함. (`blend_recovery_score`)

## 4. 교차검증 (점수 안 깎고 플래그+근거문장만)

수면은 **점수엔 안 들어가지만** 여기서 살아있음 — 객관 수면 vs 주관 감정 불일치 포착.

| 객관(수면) | 주관(감정) | status(콘솔용) | note(보호자 노출 — 비낙인 톤) |
|---|---|---|---|
| 나쁨(<40) | 좋음(≥6) | `mismatch_high_risk` | "센서 기록상 수면이 부족했던 것 같아요. 오늘은 자신을 조금 더 아껴주세요." |
| 좋음(≥70) | 나쁨(≤4) | `mismatch` | "수면은 괜찮았는데 마음이 무거운 날이네요. 그런 날도 있어요, 천천히 가요." |
| 나쁨 | 나쁨 | `agree_low` | "몸도 마음도 지친 날이에요. 무리하지 말고 충분히 쉬어주세요." |
| 그 외 | | `agree_ok` | (없음) |

> ⚠️ `note` 는 `evidence` 에 합류해 **사별 보호자에게 그대로 노출**될 수 있음 → 임상 해석
> ("숨기는 중"·"추가 관찰")은 `status` 코드로만 남기고 note 엔 비낙인 톤만(리뷰 🟡 반영).

## 5. 코드 (ai/evaluation/, 더미로 작동·테스트됨)

- `health_signal.py` — `sleep_to_score` / `activity_to_score` / `blend_recovery_score` / `cross_check` / `health_signal`(묶음). 순수함수.
- `health_adapter.py` — Health Connect `readRecords`(Steps·SleepSession) JSON → `{steps, sleep_hours}` 변환 + 더미.
- `recovery_signal.py` `compute_recovery_signal()` — `sleep_score`/`sleep_hours`/`steps` optional 인자 추가. **활동(steps)** 들어오면 `blend`로 index 교체, **수면**은 `cross_check`·표시만(점수 제외). 안 들어오면 기존과 동일.
- 테스트: `tests/test_health_signal.py`, `tests/test_health_adapter.py`.
