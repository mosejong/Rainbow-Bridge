# 오디오 후처리 가이드 (속도 조절 · BGM 합성)

> 2026-06-09 · 정환주. TTS 결과물(Qwen3 보이스, [PROTOTYPE_VOICE.md](PROTOTYPE_VOICE.md))을
> **재합성 없이** ffmpeg로 후처리하는 레시피. 음성 톤은 그대로 두고 속도·배경음만 조정.
>
> 전제: `ffmpeg` 설치됨(PATH). 작업 파일은 `Downloads/`에서 진행했고, git 미포함(음성/믹스).

## 1. 말 속도 늦추기 (`atempo`)

다이얼 `pace=slow`로도 말이 빠를 때(특히 #5 woman) 음정 유지하며 전체 속도를 늦춘다.

```bash
# 0.9 = 90% 속도(살짝 느림). 더 느리게는 0.85 / 0.8 / 0.75.
ffmpeg -y -i "audio (12).wav" -filter:a "atempo=0.9" "audio12_slow90.wav"
```

- `atempo`는 **음정을 유지**하며 길이만 늘림(0.5~2.0 범위).
- ⚠️ 너무 늘이면(0.8 이하) "놀리는/늘어지는 말투"가 됨 → **0.85~0.9 권장**.
- 다이얼의 `pace=slow`나 Google `speaking_rate`는 효과가 약함 → 이 후처리가 가장 확실.
- 근본적으로 호흡을 고르려면 문장에 쉼표·마침표를 넣어 **재합성**하는 편이 더 자연스럽다.

## 2. 음성 + BGM 합성 (인트로 → 낭독 → 페이드아웃)

BGM이 먼저 잔잔히 깔리다 음성이 들어오고, 음성 중엔 BGM이 작아지고, 끝에 페이드아웃.

### 최종 레시피 (boy/girl, 세종님 확인 반영)

```bash
# 입력: bgm_1st.mp3 + 음성wav  →  출력: *_with_bgm.mp3 (14초)
ffmpeg -y -ss 30 -t 14 -i "bgm_1st.mp3" -i "audio (10).wav" -filter_complex \
"[0:a]volume='if(lt(t,3.5),0.3,if(lt(t,4.5),0.3-0.15*(t-3.5),0.15))':eval=frame,\
afade=t=in:st=0:d=0.5,afade=t=out:st=12:d=2,aformat=channel_layouts=stereo[bg];\
[1:a]afade=t=in:st=0:d=0.3,aformat=channel_layouts=stereo,adelay=4000|4000[vo];\
[bg][vo]amix=inputs=2:duration=longest:normalize=0[out]" \
-map "[out]" "boy_with_bgm.mp3"
```

음성만 바꿔 girl도 동일하게: `audio (6).wav` → `girl_with_bgm.mp3`.

### 파라미터 의미 (조정 포인트)

| 부분 | 의미 | 조정 |
|------|------|------|
| `-ss 30 -t 14` | BGM을 **30초 지점부터 14초** 사용 | 아래 ⚠️ 참고 |
| `volume ... 0.3 ... 0.15` | 인트로 BGM **0.3**, 음성 중·여운 **0.15** | 전체 키우려면 둘 다 ↑ |
| `lt(t,3.5) … lt(t,4.5)` | 3.5~4.5초 **1초에 걸쳐 부드럽게** 줄임(ducking) | 급변 방지 — 그대로 둘 것 |
| `adelay=4000` | 음성이 **4초 뒤** 시작(= BGM 인트로 길이) | 1500~5000 |
| `afade t=in d=0.5` | BGM 시작 0.5초 페이드인 | |
| `[1:a]afade t=in d=0.3` | 음성도 0.3초 페이드인(갑툭튀 방지) | |
| `afade t=out st=12 d=2` | 12초부터 2초 페이드아웃 | `-t`와 같이 조정 |
| `amix ... normalize=0` | **필수** — 안 주면 amix가 입력 수로 나눠 소리 작아짐 | |

### ⚠️ 핵심 교훈 — "초반 BGM이 서서히 커지다 갑자기 꽝"

`bgm_1st.mp3`는 **앞부분이 페이드인(서서히 커지는) 곡**이라, 게인(`volume`)을
아무리 올려도 그 곡선 자체가 안 없어진다.
→ 해결: **조용한 인트로를 버리고 음량이 일정한 구간(`-ss 30`)부터 잘라** 쓴다.
그러면 처음부터 음량이 균일해진다. (다른 곡 쓸 땐 균일 구간을 먼저 찾을 것.)

### 음량 검증법

구간별 평균 음량을 재서 균일한지(급변 없는지) 확인:

```bash
# 0~3초(BGM 인트로)와 5~9초(음성 구간) mean_volume 비교 — 차이 작으면 균일
ffmpeg -i "boy_with_bgm.mp3" -t 3 -af volumedetect -f null - 2>&1 | grep mean_volume
ffmpeg -ss 5 -i "boy_with_bgm.mp3" -t 4 -af volumedetect -f null - 2>&1 | grep mean_volume
```

최종본 기준 인트로 ≈ -20dB, 음성 구간과 차이 1~2dB로 평탄함.

## 3. 두 음성 겹쳐 믹스 (참고)

서로 다른 두 음성을 동시에 겹칠 때(볼륨 균일화 포함):

```bash
ffmpeg -y -i a.mp3 -i b.wav -filter_complex \
"amix=inputs=2:duration=longest:normalize=0,dynaudnorm" out.mp3
```
