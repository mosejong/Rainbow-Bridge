# PERSO 아바타 발화 — 핸드오프 (`ai/tts/`)

> 작성: 정환주 · 2026-06-05 · 상태: **초안(미검증)** — 장민수 API 테스트로 확정 필요.
> 방향 전환 기록: 추모 메시지 낭독을 **Google TTS 단독 교체** 시도 → **폐기**(PERSO 단독 TTS 없음, [../../docs/PERSO_STRATEGY.md](../../docs/PERSO_STRATEGY.md)) → **PERSO AI Human 아바타 낭독 MP4를 주력, Google TTS는 안정성 폴백**으로 전환.
> 정환주 역할 = **보조**. 아바타 어댑터·PERSO 호출 구현은 **장민수**, 성격 필드 추가는 **모세종/민경이**. 이 문서는 그 두 쪽에 넘기는 spec·매핑 초안입니다.

---

## 0. 우선순위 — 목소리 = 감정 UX (높음)

- 발표 감정 품질의 핵심이 **목소리**라 우선순위를 높게 둠.
- **이중 트랙**: 주력 = **PERSO 아바타 내장 TTS**(감정 품질, 후보 검증 대상) / 폴백 = **Google TTS**(안정성·상시 동작, 기존 `voiced_url` 경로 그대로).
- 발표는 PERSO를 우선 보여주되, 실패·오프라인 대비 Google 폴백을 항상 살려둠.

## 1. 성격 필드 제안 spec (→ 모세종 백엔드 / 민경이 프론트)

매핑을 단순화하려면 **선택형(고정 선택지)** 권장 — 자유텍스트면 voiceId 매핑이 불가.

- 펫 등록([backend/app/schemas/pet.py](../../backend/app/schemas/pet.py), [frontend/src/pages/ProfilePage.jsx](../../frontend/src/pages/ProfilePage.jsx))에 `personality` 추가.
- 타입: 단일 선택(enum/Literal). **값 확정은 팀 합의 필요** — 아래는 초안.
  - `발랄` / `애교` / `성숙` / `듬직` (+ 미선택 시 `default`)
- 기존 펫 데이터엔 성격이 없으므로 **Optional + default 처리** 필요(구펫 호환).

> 현재 펫 입력값: 이름 / 종(강아지·고양이·기타) / 기간 / 추억키워드 / 사진. **성격·나이 없음** → 신규 추가 대상.

## 2. 성격 → PERSO 아바타/voiceId 매핑 테이블 (초안 → 장민수)

⚠️ **실제 voiceId·아바타 이름은 미검증.** PERSO 스튜디오 GUI 표기와 API 파라미터 값이 같은지 장민수 테스트로 확인 후 확정.

| 성격 | 아바타/목소리(후보) | 톤 의도 | 비고 |
|------|--------------------|---------|------|
| 발랄 | 채희 | 밝고 따뜻한 여성 | 채희 = "Best Fit for avatar" |
| 애교 | (부드럽고 다정한 여성, **후보 청취 필요**) | 다정 | 한채영 후보 |
| 성숙 | 박준호 | 차분하고 낮은 남성 | |
| 듬직 | 한건우 | 안정·담담 남성 | |
| **default** | 채희 | 따뜻 | 미선택 기본값 |

- 추모 톤 **발화 속도 0.85~0.90**(느리게) 권장 — PERSO 슬라이더 적용 확인.
- 코드화 시(장민수): `PERSO_VOICE_MAP[personality] -> {avatar, voiceId, speed}` 테이블 조회.

## 3. 검증 항목 (선행 — 장민수, 정환주 청취 병행)

1. **API 실호출**: spaceSeq=**555198**(팀5)에서 아바타+TTS API가 실제로 MP4를 반환하는가? (enterprise 제한 여부 — 한국어 아바타는 일반 플랜으로 보이나 미검증)
2. **GUI 이름 ↔ API 값 일치**: 채희/한채영/박준호/한건우 표기가 API 파라미터와 동일한가?
3. **감정 품질 청취**(정환주 보조): 후보들을 추모 감정 톤 기준으로 듣고 성격별 적합도 평가 → §2 테이블 확정에 반영. (클로드는 mp3 청취 불가 → 사람 청취 필수)

검증 통과 후 [../../docs/PERSO_STRATEGY.md](../../docs/PERSO_STRATEGY.md)에 아바타 낭독 경로 반영(장민수·모세종).

### 3.1 감정 품질 청취 평가표 (정환주 준비 — 음성 나오면 채점)

> 각 후보 음성에 **동일한 추모 문장**을 읽혀(예: "봄이야, 너와 산책하던 아침이 참 따뜻했어. 잘 가, 고마웠어.") 1~5점으로 채점. 합계 높은 순으로 성격 슬롯 배정.

| 평가 항목 | 설명 | 가중치 |
|------|------|------|
| 따뜻함 | 위로 톤·다정함 (추모 핵심) | ×2 |
| 자연스러움 | 한국어 억양·발음 부자연스러움 없는가 | ×1 |
| 차분함 | 너무 밝거나 들뜨지 않고 잔잔한가 | ×1 |
| 또렷함 | 발음 명료·알아듣기 쉬운가 | ×1 |
| 섬뜩하지 않음 | 기계음·과한 연기로 불쾌하지 않은가 | ×1 |

| 후보(아바타/voice) | 따뜻 | 자연 | 차분 | 또렷 | 비섬뜩 | 합계 | 적합 성격 |
|------|------|------|------|------|------|------|------|
| 채희 |  |  |  |  |  |  |  |
| 한채영 |  |  |  |  |  |  |  |
| 박준호 |  |  |  |  |  |  |  |
| 한건우 |  |  |  |  |  |  |  |

- 속도 0.85·0.90 둘 다 들어보고 추모 톤에 맞는 쪽 메모.
- 채점 결과 → §2 매핑 테이블 확정에 반영.
- ⚠️ 클로드는 mp3 청취 불가 → **사람(정환주/장민수) 직접 청취 필수.**
